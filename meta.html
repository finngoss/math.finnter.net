metamathematame

This page exists to keep track of how I run my math website, math.finnter.net.

Domain + Hosting

The basics behind how I run my website.

  • I register my domain 'finnter.net' with DreamHost for $19.99/year. This is the only part of the entire process that costs anything.
  • All of my source code lives in the main branch of this GitHub repository.
  • I use GitHub Pages configured with a custom domain name. I set my DNS records in Dreamhost to use GitHub's DNS records, as well as a CNAME record to create the 'math' subdomain.

Personal Development Environment

I am slowly but surely becoming more efficient and powerful with Linux!

  • Currently I'm running Arch Linux with the CachyOS distribution.
  • I use the Hyprland window manager and Wayland compositor to build the desktop environment on my Framework laptop 13.
  • I use Neovim as my code editor/development environment. Because I'm a beginner, I've only been tinkering around with the Kickstart configuration.
  • I am also new to the Lua language, which both Hyprland and Neovim configurations use. I started with this short tutorial which links to more robust resources.
  • Whenever I want to publish changes to my website, I just run commit my local directory for 'math.finnter.net' to GitHub and within minutes the changes are online!

Blog Architecture

My math blog is a little more complicated than the text editor + push to GitHub combination I use for the other pages.

  • I write most of my markdown and LaTeX notes inside of Obsidian because I love how easy it is to contextually relate posts with using its [[wikilinks]].
  • I then use the static-site generator Quartz to transform the markdown files in my Obsidian repository into html with working wikilinks, backlinks, and graph view.
  • Some of my posts require more than markdown, as I often enjoy including computations or graphs while exploring a new topic. For this, I use the Quarto publishing system (no relation to Quartz, funnily enough. Quarto runs .qmd files, which are like markdown files only they also give you the ability to run blocks of code like a Jupyter notebook.
  • Currently, I only use Quarto when I want to include SageMath computations or graphics. SageMath is an open-source computer algebra system that uses a python based language.
  • Quarto renders .qmd files into regular .md and image files, which I then move into the obsidian repository that Quartz draws from. Quarto also allows you to render directly into html, but I wanted my more computational notes to stay consistent with the regular markdown ones and have the same wikilink capabilities.

Command Cheatsheet

A list of commands I use to preview and push my posts.

Preview of markdown only posts with Quartz:
npx quartz build --serve, run from inside of the quartz folder in my repository.
Preview of computational posts with Quarto:
quarto preview [post].qmd
Render a computational post into plain markdown with Quarto:
quarto render [post].qmd
Quick script to move rendered Quarto markdowns into the content source for Quartz:
./publish.sh [post]
Stage + commit + push to GitHub:
git add . && git commit -m "msg" && git push
